# 🚁 Fast2026 - 中控杯

本项目旨在从零自主搭建一台具备自动导航与目标抓取能力的空中机器人（无人机），涵盖机械设计、硬件装配、ROS底层驱动与上层状态机算法。

---

## 🎯 项目核心任务分解

### 1. 🛠️ 无人机基础装配与试飞
- [ ] 根据装配文档（见 `docs/` 目录）完成飞机底盘拼装。
- [ ] 飞控参数配置，实现稳定可靠的手动遥控飞行。

### 2. 📡 机载设备部署与自动导航
- [ ] 挂载激光雷达与机载电脑。
- [ ] 烧录现有程序与环境配置。
- [ ] 实现无人机点到点的自主导航飞行。

### 3. ⚙️ 机械夹爪自主设计
- [ ] 根据抓取目标物，完成夹爪的 3D CAD 建模。

### 4. 🔌 夹爪舵机驱动开发
- [ ] 编写底层的舵机控制代码。
- [ ] 封装为 ROS 节点，实现通过 ROS 话题/服务控制夹爪开闭。

### 5. 🧠 系统集成与总控状态机
- [ ] 将夹爪节点接入现有 ROS 导航工作空间。
- [ ] 编写总控状态机

## 💻 团队协作指南 (Git 工作流)

为了避免文件冲突和代码丢失，请团队成员严格遵循以下步骤：

### 📥 1. 初次下载仓库 (Clone)
如果你是第一次参与项目，请打开终端（或 CMD）将云端代码下载到本地：
```bash
git clone https://github.com/你的用户名/Fast2026.git
```

### 🔄 2. 每日干活铁律 (Pull)
**⚠️ 警告：每次开始修改文件或写代码前，第一件事永远是更新代码！**
在 `Fast2026` 文件夹下打开终端，运行：
```bash
git pull
```

### 📤 3. 提交你的工作成果 (Add / Commit / Push)
当你在本地完成了某个文档修改、CAD模型更新或代码编写后，执行以下“三板斧”上传到云端：

```bash
# 第一步：把所有修改过、新增的文件暂存起来 (注意 add 后面的空格和点)
git add .

# 第二步：写下本次更新的备注留言 (请简明扼要，如"新增夹爪V1版打印文件")
git commit -m "在这里写你的修改备注"

# 第三步：推送到云端仓库
git push
```

---
**💡 小贴士：**
* 机械组同学提交 CAD 模型时，请**务必同时导出 STL/STEP 格式**，方便未安装 CAD 软件的同学预览。
* 代码组同学在 `ros_ws/` 中写代码时，**绝对不要**把 `build/` 和 `devel/` 文件夹传上来（项目已配置好 `.gitignore` 自动屏蔽）。
```
